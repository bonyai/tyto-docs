Tyto gives you sandboxed, remote dev environments — called **nests** — that you can create, run commands in, transfer files to and from, expose ports from, and snapshot or fork on demand.

Every nest is a full Linux VM you control over a REST API: start it from a template, run commands in a managed session or an interactive shell, upload and download files under `/home/tyto`, share a running port with a preview URL, and snapshot its disk state so you can restore or fork it later.

You can talk to Tyto directly over HTTPS, through the TypeScript or Python SDK, or from any MCP-compatible client (Claude Desktop, Claude Code, Cursor, …) using the Tyto MCP server.
